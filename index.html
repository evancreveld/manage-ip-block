Subnetting made easy

Every time you refresh this page, it generates a new IP address with a classful subnet mask. The page displays the subnet mask using three different representations.

  1. CIDR notation - the number of bits dedicated to the network portion of the IP address appears with a slash separating it from the IP address.
  2. Binary representation - IPv4 address and subnet masks are both 32 bits long. The binary subnet mask exhibits the bits used for the network portion of the IP address as '1's and the bits remaining for the host portion of the address as '0's.
  3. Dotted decimal notation - commonly found while configuring or displaying IP information of a networking interface card. The decimal notation is easier to work with than a binary mask.

CIDR, binary, and decimal are all the same when the subnet mask is concerned. Networking devices use the subnet mask to identify the subnet (the group of IPs) to which a specific IP address belongs.

Use the slider to modify the subnet mask and explore the result.

Moving the slider to the right of the IP class boundary creates subnetworks. The more bits you add to the mask, the more subnets you make, each with less IP address in it.

Moving the slider to the left of the IP class boundary creates supernetworks, also known as aggregated IP blocks. Routers use supernetworks for more efficient routing.
